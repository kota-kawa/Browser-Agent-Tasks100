# Browser-Agent-Tasks100

ブラウザ操作を評価するためのタスクベンチマークです。

## コンセプト

- **実運用に近いブラウザ操作**を、短い指示文で評価することを目的とします。
- タスクは「開始URL」「意図」「成功条件」を持ち、エージェントの**理解・探索・操作・結果確認**までを測ります。

## このベンチマークで評価されるポイント

- **指示理解**: `intent` の解釈精度
- **探索と遷移**: 目的ページへの到達力
- **操作の正確性**: フォーム入力・クリック・選択などの正確さ
- **完了確認**: `success_criteria` に沿った結果判定
- **効率**: 所要時間や操作手数（任意で記録）

## ベンチマークデータの内容

`tasks.jsonl` には、ブラウザ操作タスクが JSON Lines 形式で格納されています。
各タスクは以下の情報を持ちます。

- 開始URL（タスク開始時点のWebページ）
- 指示文（エージェントに与えるゴール）
- 成功条件（人間または自動評価が確認すべきチェックポイント）
- メタデータ（カテゴリ・難易度など）

## データ形式

各行が1タスクのJSONです。

- `task_id`: 連番ID
- `category`: カテゴリ名
- `start_url`: 開始URL
- `intent`: 実行してほしい指示文
- `difficulty`: 想定難易度 (`easy` / `medium` / `hard`)
- `success_criteria`: 成功条件のチェックポイント

タスク行の例:

```json
{"task_id": 1, "category": "search", "start_url": "https://example.com", "intent": "サイト内で...を探す", "difficulty": "easy", "success_criteria": ["..."]}
```

## 使い方（クイックスタート）

1. `tasks.jsonl` から対象タスクを選びます（全件・難易度別・カテゴリ別など）。
2. 各タスクの `start_url` をブラウザで開きます。
3. `intent` をエージェントに与えて操作させます。
4. 結果が `success_criteria` を満たすか判定します。
5. 成否・時間・手数などを記録します。

## ベンチマークの使い方（詳細）

このリポジトリは、ブラウザ操作エージェントを評価するための**タスク一覧（`tasks.jsonl`）**を提供します。
以下では、評価の準備から実行、スコアリングまでの具体的な手順を示します。

### 1. 評価対象エージェントの準備

ベンチマークは、**任意のブラウザ自動化エージェント**（自作・既存問わず）に対して実行できます。
必要なのは以下だけです。

- 指示文（`intent`）を入力できる仕組み
- ブラウザを操作し、タスク達成の可否を判定できる仕組み

### 2. タスクの選定と実行

評価手順の基本フローは次の通りです。

1. `tasks.jsonl` から評価対象のタスク集合を選ぶ  
   - 例: 全件、特定カテゴリのみ、難易度別など
2. 各タスクに対して:
   1. `start_url` をブラウザで開く
   2. `intent` をエージェントに与える
   3. エージェントが操作を完了するまで待機
   4. 結果が `success_criteria` を満たすか確認

### 3. 成功判定（スコアリング）

各タスクについて、次のように記録します。

- **Success / Fail**: 成功条件を満たしたか
- **Time**: タスク完了までの時間
- **Steps**: 実行したアクション数（任意）
- **Error type**: 失敗した場合の原因（任意）

### 4. 結果の集計と比較

タスク集合全体に対して、以下のような指標で比較できます。

- 成功率（成功タスク数 / 実行タスク数）
- 難易度別成功率（easy/medium/hard）
- カテゴリ別成功率
- 平均所要時間

これらを複数エージェントで比較することで、性能差を定量的に把握できます。

## 参考

- `tasks.jsonl` を見て、実際のタスクの粒度や指示文の書き方を把握すると運用しやすくなります。
- 評価用の結果フォーマット（CSV/JSONなど）を固定しておくと、複数エージェント間の比較が容易です。
